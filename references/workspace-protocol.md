# Workspace Protocol Reference

Deep dive into pnpm's `workspace:*` protocol and how to handle it for publishing.

## What is `workspace:*`?

pnpm workspaces use the `workspace:` protocol for internal dependencies:

```json
{
  "dependencies": {
    "@scope/shared": "workspace:*",
    "@scope/utils": "workspace:^"
  }
}
```

During development, these resolve to the local workspace package — no npm
resolution needed.

## Protocol Variants

| Protocol | Meaning | Resolved To |
|----------|---------|-------------|
| `workspace:*` | Any version (local) | Exact version (e.g., `0.2.16`) |
| `workspace:^` | Compatible version (local) | Caret range (e.g., `^0.2.16`) |
| `workspace:~` | Approximate version (local) | Tilde range (e.g., `~0.2.16`) |

## The Publishing Problem

`workspace:*` cannot be published to npm. Consumers can't resolve it:

```
npm ERR! No matching version found for workspace:*
```

## Solution: Replace → Publish → Restore

### 1. Replace (`prepare-publish.ts`)

Before publishing, replace workspace references with real versions:

```typescript
// Before
{ "@scope/shared": "workspace:*" }

// After
{ "@scope/shared": "0.2.16" }
```

The script:
1. Reads all publishable `package.json` files
2. Scans `dependencies`, `devDependencies`, `peerDependencies`
3. Replaces any `workspace:*` or `workspace:^` with the resolved version
4. Creates `.backup` files to preserve the originals

### 2. Validate (`validate-no-workspace-protocol.ts`)

Safety gate — ensures no workspace references leaked:

```typescript
// Scan all package.json files
for (const pkgPath of publishablePackages) {
  const pkg = JSON.parse(readFileSync(pkgPath, 'utf8'));
  for (const depType of ['dependencies', 'devDependencies', 'peerDependencies']) {
    for (const [name, version] of Object.entries(pkg[depType] || {})) {
      if (String(version).startsWith('workspace:')) {
        throw new Error(`${pkgPath}: ${name} still has workspace protocol: ${version}`);
      }
    }
  }
}
```

### 3. Publish

```bash
npm publish  # or pnpm publish
```

### 4. Restore (`restore-packages.ts`)

After publishing, restore the original workspace references:

```typescript
// Find all .backup files and restore them
for (const backup of backupFiles) {
  const original = backup.replace('.backup', '');
  copyFileSync(backup, original);
  unlinkSync(backup);
}
```

## Backup Strategy

`prepare-publish.ts` creates `.backup` files alongside each modified `package.json`:

```
packages/cli/package.json         ← modified (versions replaced)
packages/cli/package.json.backup  ← original (workspace:* preserved)
```

This ensures safe restoration even if publishing fails partway through.

## Which Packages Need Replacement?

Only packages being published to npm need workspace protocol replacement.
Packages that are only used internally (never published) can keep `workspace:*`.

Typical publishable packages:
- CLI packages (`packages/cli`)
- SDK packages (`packages/sdk`, `packages/shared`)
- Platform packages (generated by the publish pipeline)

## Edge Cases

### `workspace:^` resolution

```json
// Before (in package.json)
{ "@scope/shared": "workspace:^" }

// After (resolved with caret range)
{ "@scope/shared": "^0.2.16" }
```

### devDependencies with workspace protocol

`devDependencies` with workspace protocol also need replacement if the package
is published. npm publishes the full `package.json` including devDependencies.

However, since npm doesn't install devDependencies for consumers, this is
lower risk. Still, replace them for correctness.

### Circular workspace dependencies

If packages have circular workspace dependencies (A depends on B, B depends on A),
ensure the replacement script handles both in a single pass, not sequentially.

## Integration with Version Sync

The typical publish flow:

```
sync-versions     → All packages get same version
       ↓
prepare-publish   → workspace:* replaced with real versions
       ↓
validate          → Confirm no workspace: references remain
       ↓
publish           → Packages published to npm
       ↓
restore-packages  → workspace:* references restored
```
